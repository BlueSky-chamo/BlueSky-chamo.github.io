SVN学习之路（1）

发表于 2017-03-30 | 分类于 SVN | 热度 ℃
字数统计 | 阅读时长

前言

SVN是一个自由/开源的版本控制系统，一组文件存放在中心版本库，记录每一次文件和目录的修改，Subversion允许把数据恢复到早期版本，或是检查数据修改的历史，Subversion可以通过网络访问它的版本库，从而使用户在不同的电脑上进行操作。

SVN环境搭建

系统环境：windows

SVN服务器搭建

SVN服务器下载地址，选择VisualSVN server 服务端下载，选择一路next安装即可。

SVN客户端搭建

SVN客户端下载地址，选择TortoiseSVN客户端下载，安装时将command项选择安装，其他一路next即可。

SVN服务器初始配置

建立版本库

左侧Repositories项右键选择新建仓库Repositories,取名为myRepositories,依次选择empty Repositories,所有版本用户具有读/写权限。

建立用户

左侧Users项右键建立新用户，填写用户名和密码。然后选择Add将新建的用户名加入到用户中。

建立用户组

左侧Group右键建立新用户组，比如为Developers,然后Add将添加的用户加入该用户组。

用户组设置权限

左侧新建的仓库myRepositories右键选择Properties，在Security选项下，选择Add选中用户，赋予读写权限即可。

SVN客户端初始配置

连接SVN服务器Checkout

在新建的工作目录下，比如我的为work,右键-SVN Checkout,会出现一个弹框，需要填写URL of repositories。在SVN服务器端上，新建的仓库myRepositories下右键新建目录worktest,右键worktest选择copy URL,将该URL粘贴到上面，最后输入用户名和密码即可。此时服务器端的worktest即与本地的work建立起联系。

注意事项：
Checkou后本地work目录会产生一个.svn隐藏目录，这个隐藏目录记录着两项关键信息：工作文件的基准版本和一个本地副本最后更新的时间戳，不要手动修改或者删除这个.svn隐藏目录和里面的文件。
（windows下再文件夹中的工具选项，选择查看，选中显示隐藏文件）

#SVN菜单项#

SVN Update

从版本库中获取最新的一份工作拷贝，关于工作拷贝和版本库的关系，参见下面的文档。

SVN Commit

把工作拷贝所做的修改提交到版本库中

Show log

显示当前文件(夹)的所有修改历史。SVN支持文件以及文件夹独立的版本追溯。

Repo-Browser

查看当前版本库，这是TortoiseSVN查看版本库的入口，通过这个菜单项，可以进入配置库的资源管理器，然后就可以对配置库的文件夹进行各种管理，相当于我们打开我的电脑进行文件管理一样。

Revision Graph

查看当前项目或文件的修订历史图示。如果项目比较大型的话，一般会建多个分支，并且多个里程碑（稳定版本发布），通过这里就可以看到项目的全貌。

Resolved

如果当前工作拷贝和版本库上的有冲突，不能自动合并到一起，那么当你提交修改的时候，tortoisesvn就会提示你存在冲突，这时候你就可以通过这个菜单项来解决冲突。

冲突的解决有两种，一种是保留某一份拷贝，例如使用配置库覆盖当前工作拷贝，或者反过来。另一种是手动解决冲突，对于文本文件，可以使用tortoiseSVN自带的工具，它会列出存在冲突的地方

Update to Revision

从版本库中获取某一个历史版本。这个功能主要是方便查看历史版本用，而不是回滚版本。注意：获取下来之后，对这个文件不建议进行任何操作。如果你做了修改，那么当你提交的时候SVN会提示你，当前版本已失效（即不是最新版本），无法提交，需要先update一下。

Revert

如果对工作拷贝做了一些修改，但是又不想要了，可以使用这个选项把所做的修改撤销

Cleanup

如果当前工作拷贝有任何问题的话，可以使用这个选项进行修正。例如，有些文件原来是版本控制的，但是你没有通过tortoiseSVN就直接删除了，但是tortoiseSVN还是保留着原来的信息（每个文件夹下都有一个.svn的隐藏文件夹，存放着当前文件夹下所有文件夹的版本信息）所以这就会产生一些冲突。可以使用cleanup来清理一下。

GetLock/ReleaseLock

如果不想别人修改某个文件的话，那么可以把这个文件进行加锁，这样可以保证只有你对这个文件有修改权。除非你释放了锁，否则别人不可能提交任何修改到配置库中

Branch/tag

Branch是分支的意思。例如当在设计一个东西的时候，不同的人有不同的实现，但是没有经过实践检验，谁也不想直接覆盖掉其他人的设计，所以可以引出不同的分支。将来如果需要，可以将这些分支进行合并。

tag是打标签的意思。通常当项目开发到一定程度，已经可以稳定运行的时候，可以对其打上一个标签，作为稳定版。将来可以方便的找到某个特定的版本（当然我们也可以使用版本号来查找，但是数字毕竟不方便）

SVN对于分支和标签都是采用类似Linux下硬链接的方式（同一个文件可以存在两个地方，删除一个不会影响另一个，所做修改会影响另一个），来管理文件的，而不是简单的复制一份文件的拷贝，所以不会有浪费存储空间的问题存在。

Switch

切换到不同分支

Merge

合并当前的工作拷贝和版本库上相应的文件

Export

当我们需要发布一个稳定版本时，就可以使用这个功能将整个工程导出到某个文件夹，新的文件夹将不会包含任何版本信息了。

Relocate

当版本库发生转移的时候就需要用到这个功能。例如我原先的版本库是建在U盘上的，现在转移到（复制整个配置库文件夹）开发服务器上，使用https代替文件系统的访问。因此就需要将原来的工作拷贝的目标版本库重新定位到开发服务器上。

Add

将没有被版本控制的文件添加到版本库中，注意：Add操作不会自动提交到配置库中，必须再次commit。也可以在上层文件夹直接commit来添加尚未版本控制的文件（夹）

create patch

创建补丁。如果管理员不想让任何人都随便提交修改，而是都要经过审核才能做出修改，那么其他人就可以通过创建补丁的方式，把修改信息（补丁文件）发送给管理员，管理员审核通过之后就可以使用apply patch提交这次修改了。

SVN基础操作

Checkout检出

将文件checkout到本地目录(客户端操作如上连接svn服务器) 

svn checkout path（path是服务器上的目录）  
简写：svn co

Add增加

往版本库中添加新的文件 

svn add file

选择新增文件，右键SVN菜单执行“Add“操作提交到”变更列表中”，然后右键SVN菜单执行”SVN Commit”提交到版本库中

Delete（删除）

删除文件 

svn delete path -m “delete test fle”  
简写：svn (del, remove, rm)

如果被删除的文件还未入版本库，则可以直接使用操作系统的删除操作删除该文件。如果被删除的文件已入版本库，则删除的方法如下：

选择要删除文件，右键svn菜单执行”delete”操作，然后右键svn菜单执行”SVN Commit”.

Rename（改名）

文件重命名

选中需要重命名的文件或文件夹，然后右键“TortoiseSVNàRename“，在弹出的对话框中输入新名称，点击”ok”按钮，若直接选择重命名文件 “SVN Commit”提交会报如下错误：

'F:\work\text_rename.txt' which is not part of the commit; both sides of the
move must be committed together

解决：选择文件的父目录“SVN Commit”提交，会自动删除重命名之前的文件。（重命名本质上感觉是重新产生了一个文件对象，提交父目录是可以看到有两个文件，commit后会显示删除重命名前的文件）

##Revert(还原，回滚)##
若是你编辑了工程，在没有提交的前提下，想放弃这些修改，可以直接选择TortoiseSVN→revert就可以更新到工程的最新的版本。

另外可以从日志中回滚到你需要的版本，从日志中选中你需要的版本，然后Revert to this revision，把文件恢复到某个版本，然后commit，文件就回滚成功了。回滚成功后，所有的历史还存在。例如回滚到版本4，commit之后，出现新的版本6，但是他的内容和版本4是一样的。工作拷贝里面该文件回滚到这个版本（相当于在最新版本上面不断的ctrl-z撤销，而版本号还是最新的版本号，然后你所做的修改还是相当于基于最新版本所做的修改），然后就可以在这个版本上面进行修改，然后再提交。

注意：回滚只是将工作拷贝的回滚到某个版本，而不是更新版本库，只有当你回滚之后再次提交版本库才会更新，别人再次update的时候才能得到你回滚的版本。

如果选择Revert changes from this revision”，意思是“取消这个版本的修改”，即只能在当前版本上选择。若在之前版本上选择，会提示冲突，SVN会把选择权交给用户。

注意：
版本回退后，只是本地的代码文件回退到当时版本

##Update(版本更新)##
从版本库中获取最新的一份工作拷贝

更新至某一个版本的意思是获取版本库中的某一个特定的版本到工作拷贝里。如果在上面做修改，那么你是在基于旧版本的基础上做修改，当你提交的时候SVN会提示你您当前工作拷贝版本已经过期（即不是基于版本库最新版本来做修改的），这个时候你必须先update到最新版本。这个功能一般是用于获取一份只读拷贝用。

注意：只有基于最新版本上的修改才可以提交到版本库

##Move(移动文件)##
svn并没有提供move移动操作，但是可以利用rename重命名操作，在重命名前面加上路径即可。

##创建分支（branches）##

  1. trunk上右键选择Branch/Tag;

  2. 在To path输入框中输入新建分支的路径，一般是：/branches/分支名，

  3. 选择HEAD revision in the repository，为当前SVN中trunks目录下最新的代码建立分支，如果需要为制定的revision建立分支，可以进行选择; 　　

  4. 点击OK分支建立完成。打开branches目录，发现目录依然为空，没有刚才建立的分支，这是因为分支建立的操作是在服务器端完成的，当需要编辑分支时，在branches目录点击Update，刚才建立的分支就会下载下来。
